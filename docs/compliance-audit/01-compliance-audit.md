# 安全合规与审计

> 合规不是终点，是安全建设的基线。

---

## 主要合规框架

| 框架 | 适用 | 核心要求 | 审计周期 |
|------|------|---------|---------|
| **ISO 27001** | 通用 | ISMS（信息安全管理体系） | 3年认证+年度监督 |
| **SOC 2** | SaaS/云服务 | 安全/可用性/保密性/隐私 | 年度 |
| **PCI DSS** | 支付行业 | 持卡人数据保护 | 年度 |
| **等级保护 2.0** | 中国境内 | 网络安全等级保护 | 定级+备案+测评 |
| **GDPR** | 欧洲个人数据 | 数据保护/隐私 | 持续合规 |
| **HIPAA** | 美国医疗 | 健康信息保护 | 年度 |
| **FedRAMP** | 美国联邦政府 | 云服务安全 | 3年+年度 |

## 等级保护 2.0

### 安全等级

| 等级 | 适用 | 防御要求 |
|------|------|---------|
| 第一级 | 不涉及敏感信息 | 基本防护 |
| 第二级 | 一般损害 | 主动防御 |
| 第三级 | 严重损害 | 纵深防御 |
| 第四级 | 特别严重损害 | 精准防护 |

### 技术要求概览

```
安全物理环境: 温湿度/门禁/监控
安全通信网络: 网络架构/通信加密/可信验证
安全区域边界: 边界防护/入侵防范/恶意代码
安全计算环境: 身份鉴别/访问控制/安全审计
安全管理中心: 集中管控/态势感知
```

## 合规自动化工具

```bash
# OpenSCAP（自动化安全基线检查）
oscap xccdf eval --profile xccdf_org.ssgproject.content_profile_cis \
    --results results.xml --report report.html \
    /usr/share/xml/scap/ssg/content/ssg-rhel9-ds.xml

# Checkov（IaC 安全扫描）
checkov -d .
checkov --framework terraform -d terraform/
checkov --framework kustomize -d k8s/

# Prowler（AWS CIS 基线检查）
prowler aws --checks cis_1.4
prowler azure --checks cis_2.1

# ScoutSuite（多云审计）
scout aws --report-dir ./reports
```

## 审计准备清单

```
T - 6个月: 选择框架 + 差距分析
T - 5个月: 制定改进计划
T - 4个月: 编写/更新文档
T - 3个月: 技术加固实施
T - 2个月: 内部审计（预审）
T - 1个月: 整改发现项
T - 0天: 正式审计
T + 2周: 整改正式审计发现项
```
