# 个人数据跨境传输合规

## 概述

数据跨境传输是跨国企业面临的最复杂合规挑战之一——中国的数据出境规则尤其严格。本章详解个人数据跨境的三大合规路径及实操流程。

---

## 1. 数据出境判定

### 1.1 什么算"数据出境"？

```yaml
数据出境的定义 (《数据出境安全评估办法》第 3 条):

  数据处理者向境外提供在中国境内运营中收集和产生的数据:

  1. 向境外实体:
     - 直接传输到境外服务器
     - 境外实体通过 API 直接访问境内数据库
     - 发送邮件/文件到境外收件人

  2. 境外访问:
     - 境外员工远程访问境内系统
     - 境外第三方查看境内数据
     - 即使数据存储在境内 ← 这算出境！

  3. 境外处理:
     - 数据未提供, 但在境外进行处理
     - 例: SaaS 工具的境外实例处理中国用户数据

  不视为出境:
     - 在境内处理的数据不出境
     - 境内数据通过专线传输（不经过境外节点）
```

### 1.2 出境判定流程图

```python
def assess_data_export_requirement(data_type, volume, processor_type):
    """
    判断数据出境合规路径
    参考:
    - 《数据出境安全评估办法》(2022.9.1)
    - 《个人信息出境标准合同办法》(2023.6.1)
    """

    # 0. 是否涉及数据出境？
    if not is_cross_border(data_type, processor_type):
        return {"path": "NONE", "reason": "数据未出境"}

    # 1. 关键信息基础设施运营者 (CIIO) → 必须安全评估
    if processor_type == "CIIO":
        return {
            "path": "SECURITY_ASSESSMENT",
            "authority": "国家网信办",
            "timeline": "45+60 工作日",
            "reason": "CIIO 数据出境必须进行安全评估"
        }

    # 2. 重要数据 → 必须安全评估
    if data_type == "important_data":
        return {
            "path": "SECURITY_ASSESSMENT",
            "authority": "国家网信办",
            "reason": "重要数据出境必须进行安全评估"
        }

    # 3. 个人信息 → 根据规模判断
    if data_type == "personal_info":
        if volume >= 1_000_000:
            return {
                "path": "SECURITY_ASSESSMENT",
                "authority": "国家网信办",
                "reason": "处理 >100 万人个人信息"
            }
        elif volume >= 100_000:
            return {
                "path": "SECURITY_ASSESSMENT",
                "authority": "国家网信办",
                "reason": "累计向境外提供 >10 万人个人信息"
            }
        elif volume >= 10_000:
            # 敏感个人信息
            return {
                "path": "SECURITY_ASSESSMENT",
                "authority": "国家网信办",
                "reason": "累计向境外提供 >1 万人敏感个人信息"
            }
        else:
            return {
                "path": "STANDARD_CONTRACT",
                "authority": "省级网信办 (备案)",
                "timeline": "15 工作日内完成备案"
            }

    return {"path": "CONSULT_LEGAL", "reason": "需要进一步法律咨询"}
```

## 2. 安全评估流程

### 2.1 自评估 vs 申报评估

```yaml
自评估 (数据处理者自行完成):
  1. 数据出境的目的、范围、方式
  2. 境外接收方的数据保护能力
  3. 数据出境后的安全风险
  4. 遵守中国法律的保障措施
  5. 数据出境合同的合规性

  产出: 数据出境风险自评估报告

申报安全评估 (向网信办提交):
  1. 申请书
  2. 数据出境风险自评估报告
  3. 数据处理者与境外接收方签订的法律文件
  4. 安全评估工作需要的其他材料

审批流程:
  申报 → 形式审查(7日) → 受理 → 评估(45日) 
  → 复杂案件可延长(60日) → 结果通知书
```

### 2.2 法律文件要点

```markdown
# 数据出境法律文件 (合同) 必备条款
# 《数据出境安全评估申报指南 (第二版)》

## 1. 数据出境的目的、方式和范围
- 出境数据字段及类型说明
- 处理目的及必要性说明
- 数据规模 (人数/条数/GB)
- 保留期限

## 2. 境外接收方的义务
- 按约定目的、方式处理数据
- 采取与数据安全等级匹配的保护措施
- 数据泄露通知 (<24 小时内)
- 协助履行数据主体权利
- 删除/匿名化义务 (出境期限届满后)

## 3. 数据处理者的监督权利
- 定期审计权 (≥年度)
- 安全事件调查权
- 停止数据传输权

## 4. 数据主体的权利
- 知情权
- 访问权、更正权、删除权
- 投诉权 (接收方当地监管机构 + 中国网信办)

## 5. 违约责任
- 违约金计算方式
- 赔偿范围 (包括数据主体损失)
```

## 4. 实操清单

```yaml
数据出境合规操作清单:

  Step 1 - 数据盘点:
    - [ ] 识别所有涉及个人数据的系统
    - [ ] 梳理数据流: 谁/何时/向谁/传输了什么数据
    - [ ] 确认所有境外数据接收方

  Step 2 - 出境判定:
    - [ ] 是否存在"境外主体直接访问境内数据库"
    - [ ] 确认 SaaS/云服务的服务器位置
    - [ ] 确认邮件/文件分享的外发路径

  Step 3 - 路径选择:
    - [ ] CIIO? → 安全评估
    - [ ] 处理 >100 万个人信息? → 安全评估
    - [ ] 重要数据? → 安全评估
    - [ ] 其他? → SCCs 备案

  Step 4 - 安全评估:
    - [ ] 完成自评估报告
    - [ ] 签订法律文件
    - [ ] 提交申报

  Step 5 - 持续合规:
    - [ ] 定期数据流审计 (年度)
    - [ ] 接收方安全能力评估
    - [ ] 数据主体权利响应 (15 日内)
    - [ ] 出境情况变更 (重新评估)
```
