# 安全检查清单

> 安全开发与运维的实用检查清单

---

## 开发安全清单

### 通用
- [ ] 所有用户输入在输出前进行 HTML 实体编码
- [ ] 所有数据库查询使用参数化查询（Prepared Statements）
- [ ] 所有 API 端点验证用户权限（不只是登录状态）
- [ ] 文件上传限制类型、大小，并重命名存储
- [ ] 不信任用户提供的文件名和 Content-Type
- [ ] 系统命令执行使用参数列表形式（禁止 shell=True）
- [ ] XML 解析禁用外部实体（用 defusedxml）
- [ ] 反序列化只处理可信数据，使用 JSON 替代 Pickle
- [ ] 所有表单需要 CSRF Token 保护
- [ ] 密码存储使用 bcrypt/argon2，不使用 md5/sha1

### API 安全
- [ ] 认证：使用 JWT/OAuth，并进行签名验证
- [ ] 速率限制：防止暴力破解和 DoS
- [ ] CORS：只允许必要的来源
- [ ] 输入验证：严格校验所有参数类型和格式
- [ ] 输出脱敏：不返回密码、Token 等敏感信息
- [ ] 日志：记录操作但不记录敏感数据

### 配置安全
- [ ] 不在代码中硬编码密钥/密码（使用环境变量）
- [ ] 关闭不必要的服务和端口
- [ ] 使用 HTTPS（HTTP Strict Transport Security）
- [ ] 设置安全的 Cookie 属性（HttpOnly、Secure、SameSite）
- [ ] 配置 Content-Security-Policy 头
- [ ] 禁用目录列表
- [ ] 错误页面不展示堆栈信息

## AI 安全清单

### Agent 安全
- [ ] 系统提示词不包含敏感信息
- [ ] Agent 工具调用需要参数验证
- [ ] 高风险操作需要人工确认
- [ ] 外部内容在使用前检查隐藏指令
- [ ] 记录所有工具调用日志
- [ ] API Key 通过连接管理存储，不在对话中出现

### LLM 安全
- [ ] 输入过滤：检测和阻止 Prompt 注入
- [ ] 输出过滤：不在输出中泄露敏感信息
- [ ] 内容隔离：用户输入和系统指令明确分隔
- [ ] 最小权限：Agent 只申请必要的工具权限
- [ ] 人工审核：高风险操作需要二次确认

## 应急响应清单

### 发现安全事件时
- [ ] 确认漏洞真实存在（排除误报）
- [ ] 评估影响范围（哪些系统、哪些数据受影响）
- [ ] 评估风险等级（CVSS 评分）
- [ ] 通知相关方（安全团队、法务、管理层）
- [ ] 立即修复（代码级/配置级）
- [ ] 验证修复有效性
- [ ] 复盘文档（发生了什么、怎么发生的、如何改进）
