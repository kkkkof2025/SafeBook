# AI 安全概述

> 大模型时代的安全新挑战 —— LLM 带来了新的攻击面，也改变了原有的威胁模型。

---

## AI 安全的独特性

与传统安全相比，AI 安全有本质的不同：

| 维度 | 传统安全 | AI 安全 |
|------|----------|---------|
| 攻击面 | 代码、网络、系统 | 模型、Prompt、训练数据、向量数据库 |
| 漏洞类型 | SQLi、XSS、RCE 等 | 注入、越狱、数据投毒、幻觉利用 |
| 防御方式 | 输入验证、参数化查询、CSP | Prompt 护栏、内容过滤、模型对齐 |
| 攻击成本 | 需要技术能力 | 自然语言即可发起攻击 |
| 自动化程度 | 需要编写脚本/工具 | LLM 可以自动生成攻击 |

## AI 安全的挑战

1. **不可预测性**：LLM 是概率模型，同样的输入可能产生不同的输出
2. **自然语言攻击**：传统攻击需要技术知识，LLM 攻击只需要会说话
3. **供应链复杂**：模型训练数据、第三方 API、插件扩展都可能是攻击入口
4. **监管滞后**：AI 安全标准还在发展中

## 基本原则

在 AI 系统中应用安全，需要遵循：

1. **最小权限**：Agent/MCP 工具只给必要权限
2. **审核机制**：所有 Agent 执行的操作需要人工确认或审计
3. **内容隔离**：系统提示词和用户输入分离
4. **出口控制**：Agent 能访问的外部资源必须限制
5. **持续监控**：记录所有 LLM 调用和 Agent 操作，便于回溯
