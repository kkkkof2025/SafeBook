# 信息收集与侦察

> 在找漏洞之前，先搞清楚目标长什么样

---

## 为什么信息收集很重要

**安全测试的 80% 时间花在信息收集上**。你掌握的信息越多，找到漏洞的概率越大。

```
信息收集（80%） → 漏洞分析（15%） → 漏洞利用（5%）
```

## 信息收集框架

### 被动信息收集（不接触目标）

```
┌─────────────────────────────────────────────────────┐
│                   被动收集                            │
├─────────────────────────────────────────────────────┤
│ 搜索引擎    │ GitHub     │ 证书透明   │ DNS 记录     │
│ (Google)   │ (代码泄露)  │ (crt.sh)  │ (子域名)      │
├────────────┼────────────┼───────────┼──────────────┤
│ 企业信息    │ 社交平台    │ 招聘信息   │ Wayback      │
│ (企查查)   │ (LinkedIn) │ (技术栈)   │ (历史记录)    │
└─────────────────────────────────────────────────────┘
```

### 主动信息收集（直接接触目标）

```
┌─────────────────────────────────────────────────────┐
│                   主动收集                            │
├─────────────────────────────────────────────────────┤
│ 端口扫描    │ 目录枚举   │ 指纹识别    │ 漏洞扫描     │
│ (端口/服务) │ (隐藏路径) │ (技术栈)    │ (已知漏洞)    │
└─────────────────────────────────────────────────────┘
```

## 实战收集流程

### 第一步：域名信息

```bash
# 子域名枚举
subfinder -d example.com

# DNS 记录查询
nslookup example.com
dig example.com ANY

# 证书透明日志（查找子域名）
curl -s "https://crt.sh/?q=%.example.com&output=json"

# 历史页面（可能包含敏感信息）
curl "https://web.archive.org/cdx/search/cdx?url=*.example.com&output=text"
```

### 第二步：技术栈识别

```bash
# WhatWeb — 识别 Web 技术栈
whatweb example.com

# Wappalyzer — 浏览器插件（方便）
# 安装后访问目标网站自动识别
```

### 第三步：目录和文件枚举

```bash
# dirsearch — 目录爆破
dirsearch -u https://example.com -w /usr/share/wordlists/dirb/common.txt

# ffuf — 快速模糊测试
ffuf -u https://example.com/FUZZ -w wordlist.txt
```

### 第四步：端口和服务

```bash
# 快速扫描 TOP 1000 端口
nmap -sV -sC -T4 example.com

# 全端口扫描（慢但全面）
nmap -p- -T4 example.com
```

## 信息收集清单

### 基础信息

- [ ] 目标是什么类型的网站？（电商/社交/企业站/API）
- [ ] 目标的主要功能是什么？（登录/搜索/上传/支付）
- [ ] 目标的目标用户是谁？

### 技术信息

- [ ] Web 服务器：Nginx/Apache/IIS？
- [ ] 后端语言：PHP/Python/Java/Go/.NET？
- [ ] 前端框架：React/Vue/jQuery？
- [ ] 数据库：MySQL/PostgreSQL/MongoDB？
- [ ] 是否有 WAF/CDN？

### 攻击面

- [ ] 登录/注册页面
- [ ] 搜索功能
- [ ] 文件上传
- [ ] API 端点
- [ ] 忘记密码
- [ ] 用户资料页
- [ ] 管理后台
- [ ] 第三方集成

## 信息收集的伦理边界

- ✅ 公开信息收集：允许
- ✅ 使用公开工具扫描：允许（在授权范围内）
- ❌ 社会工程学攻击个人：需要特殊授权
- ❌ 暴力破解登录：需要明确授权

---

## 小结

- 信息收集决定了你能找到什么漏洞
- 先被动、后主动，避免打草惊蛇
- 系统和规范化地收集，不要东一榔头西一棒子
