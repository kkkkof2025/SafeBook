# 学习路径总览

> 安全不是一门可以速成的学科，但有清晰的学习路径可以让你少走弯路。

---

## 三个并行的线程

安全学习不是线性的，而是三个并行的线程：

### 线程一：理论基础
- 计算机网络、操作系统、数据库
- Web 技术栈（HTTP/HTML/JS/SQL）
- 密码学基础
- 安全方法论（威胁建模、风险分析）

### 线程二：动手实践
- 搭建靶场、运行 POC
- CTF 竞赛
- 漏洞赏金、渗透测试

### 线程三：持续跟进
- 关注 CVE/NVD
- 阅读安全博客和论文
- 参加安全会议

---

## 学习建议

按照下面的路线图走，每一步都可以在 [推荐平台](02-platforms.md) 中找到对应的资源。

> **关键原则**：不要追求一步到位，安全是积累型的技能。今天看懂一个漏洞，明天多跑一个 POC，后天修复一个 Bug，慢慢就形成体系了。
