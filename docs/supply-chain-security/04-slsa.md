# SLSA 框架

## SLSA 级别

SLSA（Supply-chain Levels for Software Artifacts）定义四个安全级别：

### SLSA 1（基础）
- 构建过程有文档记录
- 源代码受版本控制
- 适合内部工具

### SLSA 2（增强）
- 使用托管构建服务
- 构建来源可验证
- 适合开源库

### SLSA 3（强化）
- 构建来源不可抵赖
- 隔离构建环境
- 无用户自定义步骤
- 适合关键依赖

### SLSA 4（最高）
- 双人审查
- 可复现构建
- 适合安全关键组件

## 实施建议

1. 从 SLSA 2 开始，使用托管 CI/CD
2. 逐步提升构建隔离和签名
3. 对核心依赖要求 SLSA 3+
