# 蓝队日常安全运营

## 安全运营中心架构

### SOC 三层模型
```
Tier 1（一线）
  监控告警 → 初步分类 → 升级或关闭
  ⬇
Tier 2（二线）
  深度分析 → 事件确认 → 应急响应
  ⬇
Tier 3（三线）
  根因分析 → 溯源 → 反制
```

## 威胁狩猎（Threat Hunting）

### 狩猎方法论
```python
def threat_hunt():
    1. 假设（Hypothesis）
       "假设攻击者已通过钓鱼进入内网"
    2. 数据收集
       收集端点遥测、DNS 日志、网络流
    3. 模式检测
       异常进程链、异常 DNS 查询
    4. 调查
       确认 IoC → 横向移动追踪
    5. 响应
       隔离 → 取证 → 修复
```

### 常见狩猎场景

| 场景 | 检测点 | 工具 |
|------|--------|------|
| 横向移动 | 异常 RDP/SSH 连接 | Zeek + RITA |
| 数据外泄 | 大流量出站 | NetFlow |
| 权限提升 | 异常进程树 | Sysmon + EDR |
| 持久化 | 新建服务/计划任务 | Windows Event Log |
| C2 通信 | Beacon 周期性心跳 | Zeek + JA3 |

## 安全 KPI

| 指标 | 目标 | 计算方式 |
|------|------|---------|
| MTTD(发现时间) | < 1 小时 | 入侵到发现的时间 |
| MTTR(响应时间) | < 4 小时 | 发现到阻断的时间 |
| 告警误报率 | < 5% | 误报/总告警 |
| 漏洞修复时间(严重) | < 24 小时 | CVSS 9+ 的修复周期 |
| 补丁覆盖率 | > 95% | 已打补丁/需打补丁 |
