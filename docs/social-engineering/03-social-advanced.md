## 深度钓鱼：鱼叉攻击实战案例

### 案例：渗透某互联网公司

```
目标: 获取某甲方内网代码仓库权限

侦察阶段（2天）:
  ├─ 脉脉搜索目标公司员工 → 锁定运维主管 王工
  ├─ GitHub 找到王工的提交记录（含公司邮箱）
  ├─ 微博找到王工是摄影爱好者（索尼A7用户）
  └─ 发现王工最近在找摄影教程

钓鱼阶段（1天）:
  ├─ 冒充"摄影之友"编辑：
  │   "您好王老师，我们公众号最近在做一期企业摄影分享，
  │   看到了您的作品很专业。能否加个微信聊聊合作？"
  └─ 王工通过好友申请

深化阶段（3天）:
  ├─ 分享高质量摄影文章（建立信任）
  ├─ 第三天发送 "独家摄影调色预设"（含恶意宏的PSD文件）
  ├─ "王老师，这是我们签约摄影师的最新预设包，
  │   需要解压密码，方便微信说一下？"
  └─ 王工打开文件 → 上线C2

结果:
  ├─ 获取内网代码仓库只读权限
  └─ 发现 GitHub Token 明文存储在 config 文件
```

**关键成功因素**：精准的目标画像 + 足够的耐心 + 符合对方兴趣的切入点

## 反社会工程学防御

### 员工行为准则

```
🔴 永不：
  - 在电话中透露系统信息和密码
  - 让他人尾随进入无门禁区域
  - 把门禁卡借给"忘带"的同事
  - 在社交网络发布工牌/公司内景
  - 点击非预期的链接/附件

🟢 必须：
  - 任何要求提供凭证的，先电话回拨确认
  - 发现可疑 → 立即上报安全团队
  - 定期参加安全意识培训
  - 使用密码管理器（自动识别钓鱼网站）
```

### 企业级防御

| 控制措施 | 说明 | 优先级 |
|---------|------|--------|
| MFA | 所有系统强制多因素认证 | P0 |
| 模拟钓鱼 | 每月自动发送测试钓鱼邮件 | P0 |
| 门禁审计 | 刷卡记录每周审计 | P1 |
| 电话验证 | 敏感操作需电话回拨 | P1 |
| 办公区监控 | 陌生人识别+尾随告警 | P2 |
| 垃圾销毁 | 碎纸机覆盖所有纸质文档 | P2 |

## 社工库查询（仅防御用途）

> ⚠ 以下渠道仅用于组织自身数据泄露自查，禁止用于侵犯他人隐私

- **Have I Been Pwned**：检查邮箱是否在已知泄露中
- **Firefox Monitor**：Mozilla 泄露检测
- **Google Password Checkup**：Chrome 内置泄露检测
- **IntelX**：深网泄露数据（防御用）
- **DeHashed**：企业泄露监控

```python
# 企业批量自查
import requests

def check_company_leaks(domain):
    # 使用 HIBP API 检查公司邮箱
    for email in get_company_emails(domain):
        resp = requests.get(
            f"https://haveibeenpwned.com/api/v3/breachedaccount/{email}",
            headers={"hibp-api-key": "YOUR_KEY"}
        )
        if resp.status_code == 200:
            print(f"[!] {email} 已泄露: {resp.json()}")
```
