# 社会工程学概论

> 最坚固的防火墙也无法防御被"肉鸡化"的人类——社会工程学是绕过所有技术防护的最短路径。

---

## 什么是社会工程学

社会工程学（Social Engineering）是利用**人类心理弱点**而非技术漏洞获取信息或权限的攻击方式。

### 攻击链
```
信息收集 → 信任建立 → 诱导操作 → 信息窃取 → 权限利用
```

### 心理学原理

| 原理 | 说明 | 利用方式 |
|------|------|---------|
| 互惠 | 对方帮了你，你也想帮他 | 先送小礼物后提要求 |
| 承诺一致 | 人倾向于言行一致 | 先让答应小请求，再提大请求 |
| 社会认同 | 跟随大家行为 | "你们部门的张工也是这么操作的" |
| 权威 | 服从权威人物 | 冒充IT/审计/高管 |
| 喜好 | 喜欢的人更容易答应请求 | 建立共同兴趣/赞美 |
| 稀缺 | 稀缺的更有价值 | "今天不操作就过期了" |
| 紧迫 | 时间压力降低判断力 | 制造紧急事件 |
| 恐惧 | 害怕损失 | "你的账户已被入侵" |

## 常见攻击类型

### 1. 钓鱼攻击（Phishing）

```
鱼叉（Spear）-> 针对特定个人
鲸钓（Whaling）-> 针对高管
诱饵（Baiting）-> USB掉落/下载陷阱
水坑（Watering Hole）-> 感染目标常访问网站
短信（Smishing）-> 短信钓鱼
语音（Vishing）-> 电话钓鱼
```

### 2. 预文本攻击（Pretexting）

```python
# 冒充IT支持获取密码重置权限
pretext_script = """
1. 目标: 公司IT运维张三
2. 角色: 某软件供应商售后
3. 话术: "我是XX系统的售后工程师，系统监测到贵司的许可证即将过期，
          需要远程确认服务器上的授权文件。方便加个微信发一下服务器登录IP？"
4. 目标: 获取服务器SSH/VPN访问凭证
"""
```

### 3. 尾随（Tailgating / Piggybacking）

- 假装忘带门禁卡
- 扮演外卖员/快递员
- 跟随持卡员工进入（通常不会阻拦）
- 借烟/借火转移注意力进门

### 4. 电话钓鱼（Vishing）Demo

```
攻击者: "您好，我是××银行风控部，系统监测到您的银行卡在境外有一笔
         8,000美元的消费，请问是您本人操作吗？"

受害者: "啊？不是！我没有出境。"

攻击者: "那我帮您冻结账户，需要先验证身份——请告诉我您卡号后四位和
         手机验证码。"

受害者: [不知不觉交出验证信息...]
```

## 信息收集来源

| 来源 | 可获取信息 | 难度 |
|------|-----------|------|
| LinkedIn | 职位、履历、同事关系 | 低 |
| 微博/朋友圈 | 日常作息、家庭情况、兴趣爱好 | 低 |
| GitHub | 代码、邮箱、项目、公司内网信息 | 低 |
| 公司官网 | 组织架构、供应商、联系方式 | 低 |
| 招聘信息 | 技术栈、内部系统名称 | 低 |
| 暗网 | 泄露凭据、内鬼情报 | 高 |
| 垃圾桶 | 纸质文档、便签密码（Dumpster Diving） | 低 |

## 企业防护策略

### 技术防护
```yaml
邮件过滤: DMARC/DKIM/SPF 三件套
URL 沙箱: 点击链接前沙箱检测
多因素认证: 杜绝单一凭据沦陷
模拟钓鱼: 定期内部钓鱼演练
```

### 管理策略
```
✅ 安全意识培训（每季度）
✅ 钓鱼模拟测试（每月发送测试钓鱼邮件）
✅ 明确告警上报流程（发现可疑立即上报）
✅ 资产丢弃制度（碎纸机销毁文档）
✅ 访客登记制度（全程陪同、不得单独行动）
✅ 信息最小化原则（不在官网列出员工全名+职位+邮箱）
```

## 红队实战检查表

```
[ ] 目标员工 LinkedIn/脉脉 信息收集
[ ] 公司 GitHub 组织成员、仓库权限分析
[ ] 招聘信息中的技术栈分析
[ ] 伪造身份（IT/审计/供应商/HR）
[ ] 发送诱饵邮件（含宏/漏洞文档）
[ ] 电话预文本（冒充IT重置密码）
[ ] 物理渗透测试（尾随/伪装）
[ ] USB 诱饵投放（公司停车场/吸烟区）
```
